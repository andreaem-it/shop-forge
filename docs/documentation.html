ShopForge — Documentation

Version 1.8.0 — a modular WooCommerce plugin for account area, tracking, wishlist, returns, quotes and notifications.

Contents 1. Installation 2. License activation 3. Modules and core features 4. Themes (skins) 5. Colors 6. Shortcodes 7. Shipment tracking setup 8. FAQ & troubleshooting

1. Installation

  1. In WordPress admin go to Plugins → Add New → Upload Plugin.
  2. Select shopforge.zip and click Install Now, then Activate.
  3. WooCommerce must be installed and active — ShopForge shows an admin notice if it is not.
  4. A new ShopForge menu item appears in the WordPress admin sidebar.

2. License activation

  1. Go to ShopForge → License.
  2. Paste the license key you received at purchase and click Validate license.
  3. Once valid, all modules become available for activation under the Modules tab. Core features (styles, dashboard) work without a license.

3. Modules and core features

Go to ShopForge → Modules. Two sections are shown:

  • Core features — cross-cutting behavior: account area styles, shop/catalog styles, custom colors, custom dashboard. No license required.
  • Modules — self-contained features with their own account endpoint. Require an active license.
ModuleWhat it does
Shipment trackingTracking widget on the order detail page, powered by 17track.net. See section 7.
WishlistHeart button on catalog and product pages; saved products list in the account.
QuotesCustomers request a quote for a list of products; you reply from ShopForge → Quotes.
Support & ReturnsTwo-step withdrawal request flow with an automatic receipt email, compliant with EU consumer law.
Product Support (RMA)Structured repair/replacement/refund requests per order line, with message thread, refund automation, CSV export and stats.
NotificationsIn-account notification center for order, ticket, return and quote events.

Toggling a module off immediately removes its account menu entry, its endpoint, and stops loading its PHP file — zero footprint when disabled.

4. Themes (skins)

Under ShopForge → Modules → Theme you can pick one of three layouts:

  • Clean — flat, white background, bordered cards, no shadows. Matches the WooCommerce block checkout style.
  • Boxed — content in a soft gray panel, elevated cards with shadows.
  • Cards — every item is a separate rounded card with more spacing.

You can also assign a different theme per context (Cart, Checkout, Account area, Shortcodes/other pages) — leave "Global theme" to inherit the main selection.

5. Colors

Under the same Modules screen, the Colors section lets you edit the full palette (primary, hover, text, borders, backgrounds, success/warning/danger). Colors are injected as CSS variables on the front end, so any custom CSS you add using var(--shopforge-primary) etc. stays in sync.

7. Shipment tracking setup

  1. Create a free account at api.17track.net and get an API key.
  2. Go to ShopForge → Modules → Configuration and paste the key into 17track API key.
  3. Enable the Shipment tracking module.
  4. On each order's edit screen, enter the carrier name and tracking number in the "Shipment Tracking" metabox.
  5. The customer sees a live tracking widget on their order detail page in the account area.

Without an API key the module loads but the tracking widget is not shown.

8. FAQ & troubleshooting

The cart/checkout layout looks broken with my theme. The plugin's CSS uses !important on the layout-critical rules specifically to override theme interference; if you still see issues, check whether your theme injects layout CSS after ShopForge's stylesheet (inspect the load order in browser dev tools) and increase specificity in your child theme if needed.

A module doesn't appear even though I enabled it. Modules require a valid, active license. Check ShopForge → License.

I need support. Contact us through the item's comment page on CodeCanyon or via the author URL in the plugin header.
